NSObject相关理解

发表于 2017-05-02 |

NSObject

在Cocoa里有两个NSObject，一个是类，一个是协议。

NSObject的类和协议在Objective-c中存在于不同的命名空间，它们在编程语言上是不相关联的。所以可以用相同的名称来命名类和协议。

NSObject类遵循NSObject协议

协议：顾名思义，制定的一份协议，相当于预留了一份未完成的任务清单（即很多未实现的方法）。谁如果遵守这个协议，并且成为这个协议的代理。那么就可以在协议制定者需要做这件事情的时候，让代理完成剩下的任务。【注意】协议也可以继承。

Q1: NSObject是一个基类，这里为什么要使用协议，而不直接把这些方法实现在NSObject类中？

A: 因为Cocoa中不止有一个基类，NSObject协议可以理解为一个基础协议，多个基类都遵守这个基础协议，那么就相当于拥有了共同的基础方法。

常用方法

Initializing a Class

当这个类在工程中第一次出现的时候，会调用这个方法，并且只会调用一次。 当这个类的子类第一次出现的时候，并且这个类还没有出现过，会先调用父类的initialize方法，然后调用这个子类的initalie方法。

无论何时，只要把一个类或类别添加进OC的运行时，就会调用该方法。

Creating, Copying, and Deallocating Objects

使用alloc方法，在内存中开辟存储空间，并清零，将内存空间的首地址返回。

将刚刚开辟的内存空间设置上初始值和必要的配置（比如：注册方法列表等）。与alloc结合使用。

返回一个通过copyWithZone:创建的实例。NSObject类本身并没有实现copyWithZone:这个方法，这个方法的实现是在NSCopying协议里面实现的。所以，子类如果需要调用copy方法，那么子类必须遵守NSCopying协议才能正常使用。

当zoon为nil的时候，通过NSMutableCopying 协议方法 mutableCopyWithZone:创建的对象。

copy与mutableCopy

copy返回的是不可变对象，mutableCopy返回的是可变对象。

对于不可变对象：调用copy返回该对象本身，调用mutableCopy返回一个新的可变对象

对于可变对象：调用copy返回一个新的不可变对象，调用mutableCopy返回一个新的可变对象。

当对象的引用计数为0时，系统会自动调用dealloc方法，回收内存。

分配新的内存空间给一个新实例对象，这个对象发送init消息，返回一个初始化过的对象。

new和alloc/init

alloc 方法 会调用 zero方法 ，会给有关联的对象分配相邻的内存地址，方便查找，提高性能。 并且和init方法分开，可拓展性更强。

new 没有调用zero 方法，只是在任意位置开辟了内存空间。然后通过init方法进行了初始化，所以，对关联性强的对象性能没有优化。并且可扩展性不强。对要求不高的初始化可以选择new。

Describing Objects

Instance Methods

判断两个对象是否相等。

判断是否是某个类或是继承自该类的子类。

判断是否为指定类。

isKindOfClass:和isMemberOfClass:

isKindOfClass:只要判断的对象是指定类或指定类的子类。那么就返回YES

isMemberOfClass:判断的对象必须是当前指定的类，才会返回YES
